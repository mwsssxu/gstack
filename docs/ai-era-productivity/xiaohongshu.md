## 🛠️ 核心角色速览

### 产品思考者：顶级投资人思维 💡

**6 个灵魂拷问：**
1. 📊 需求证据？（不是"感兴趣"，是"会沮丧"）
2. 🕵️ 现状分析？（用户现在怎么解决？）
3. 👤 具体用户？（说名字，不说"中小企业"）
4. 🎯 最窄楔子？（最小付费版本是什么？）
5. 😲 观察意外？（用户做了什么让你惊讶？）
6. 🔮 未来适配？（3 年后更重要还是更不重要？）

### 战略规划师：CEO 战略思维 🎯

**4 种模式：**
- 🚀 扩张：无限资源会做什么？
- 🎯 精选：保持范围 + 诱人机会
- 🔒 保持：让它无懈可击
- ✂️ 缩减：最小可交付价值

### 架构设计师：架构可靠性 🏗️

**首要原则：零沉默失败！**

**强制图表：**
- 正常流程 ✅
- 输入缺失 ❌  
- 输入为空 🕳️
- 上游失败 💥

### 偏执专家：偏执审查 🔍

**8 大结构性缺陷：**
1. 🗃️ 数据安全（N+1 查询、缺失索引）
2. ⚡ 重复操作（双重提交、覆盖写入）
3. 🤖 AI 信任边界（JSON 错误、幻觉值）
4. 🔢 选项完整性（新增值未处理）
5. 🎭 条件副作用（分支不一致）
6. 🔑 写死数值（硬编码值）
7. 💀 无用代码（未使用、过时注释）
8. 📝 AI 提示问题（泄露、溢出）

### 质量专家：智能测试 🧪

**4 种模式：**
- ⚡ 快速：30 秒烟雾测试
- 📋 标准：5-15 分钟系统探索  
- 🔍 穷尽：深度测试所有问题
- 📈 回归：变更影响分析

### 发布专家：安全发布 🚀

**发布清单：**
- ✅ 测试引导（无框架？自动创建）
- ✅ 覆盖审计（路径图）
- ✅ 审查门控（仪表板控制）
- ✅ 文档自动化（协同更新）
- ✅ 版本管理（语义化自动化）
- ✅ 恢复准备（安全网）

## 🚀 行动指南：如何开始？

### Step 1️⃣ 选择一个痛点开始
- 工作审查太慢？→ 从 偏执专家 开始
- 测试覆盖不足？→ 从 质量专家 开始  
- 产品方向不清？→ 从 产品思考者 开始

### Step 2️⃣ 建立标准工作流
```
新功能:
1. 产品思考者 → 设计文档
2. 战略规划师 → 范围决策
3. 架构设计师 → 架构计划
4. 实现
5. 偏执专家 → 工作审查
6. 质量专家 → 系统测试
7. 发布专家 → 发布
```

### Step 3️⃣ 重新定义你的角色 💪
- 以前：执行者 → 现在：审查 AI 工作的人
- 以前：测试专家 → 现在：设计测试策略的人  
- 以前：产品经理 → 现在：定义问题空间的人

**你的价值不再是"做什么"，而是"决定做什么"！**
